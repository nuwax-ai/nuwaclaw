## 三、Bug 调试流程

### 场景：SSE 连接断开

```
👤 SSE 连接经常断开，怎么回事？

🤖 让我分析一下。

🔍 诊断流程：

1. 查看错误日志
   - 主进程日志：~/Library/Logs/electron-app.log
   - 渲染进程：DevTools Console

2. 常见原因：
   - 网络不稳定
   - 服务端超时配置
   - 客户端重连逻辑问题
   - Token 过期

3. 检查 sseManager.ts
   - 重连机制是否启用
   - 心跳检测是否正常
   - 超时配置是否合理

👤 帮我检查一下 sseManager

🤖 [分析代码...]

- sseManager 没有实现心跳检测

1. 添加心跳检测（每 30s）
2. 重连间隔改为指数退避（1s → 2s → 4s → 8s → max 30s）
3. 添加最大重连次数
```

## 八、常见问题处理

### SSE 连接

| 问题 | 原因 | 解决 |
|------|------|------|
| 连接断开 | 网络不稳定 | 添加重连机制 |
| 消息延迟 | 服务端队列堆积 | 优化推送逻辑 |
| Token 过期 | 无刷新机制 | 添加 Token 刷新 |

### AntV X6

| 问题 | 原因 | 解决 |
|------|------|------|
| 节点无法拖拽 | 节点未注册 | 检查节点注册 |
| 连线错位 | 锚点位置不对 | 调整 anchor |
| 画布卡顿 | 节点过多 | 使用虚拟化 |

### UmiJS

| 问题 | 原因 | 解决 |
|------|------|------|
| 路由 404 | 动态路由未配置 | 检查路由配置 |
| model 失效 | namespace 重复 | 检查 model 注册 |
| 插件不生效 | 插件顺序错误 | 调整 plugins 顺序 |
