# Agent Autonomy Levels

> 定义 Agent 的自主能力等级

---

## 自主能力演进

| 等级 | 描述 | 人类介入 |
|------|------|----------|
| **L1** | 验证状态 + 重现问题 | 持续 |
| **L2** | 录制故障视频 + 实施修复 | 需要 approve |
| **L3** | 验证修复（运行应用） | 需要 approve |
| **L4** | 打开 PR | 需要 approve |
| **L5** | 回应反馈 | 自动 |
| **L6** | 检测构建故障 | 自动 |
| **L7** | 仅在需要判断时交由人工 | 自动 |
| **L8** | 自动合并（通过 CI） | 自动 |
| **L9** | 自主学习 + 优化 harness | 定期 review |

---

## 默认等级

### 推荐：L2-L4

大多数项目使用 **L2-L4**：
- L2: Agent 发现问题并修复
- L3: Agent 验证修复
- L4: Agent 打开 PR，人类 review

### 高信任团队：L5-L6

信任度高的团队可以使用 L5-L6：
- Agent 自动合并通过 CI 的 PR
- Agent 回应简单反馈

### 实验性：L7-L9

需要成熟的 harness 和完善的测试：
- 仅在需要判断时交由人工
- 自动合并
- 自主学习

---

## 等级启用

在 `harness/feedback/state/state.json` 中配置：

```json
{
  "autonomy": {
    "level": 3,
    "requireApprovalFor": ["delete", "network_request"],
    "autoMergeOnCI": false
  }
}
```

---

## 各等级行为

### L1: 助手
```
人类：帮我修这个 bug
Agent：分析后告诉你需要怎么做
人类：执行修复
```

### L2: 执行者
```
人类：帮我修这个 bug
Agent：分析 → 修复 → 验证 → 报告结果
人类：approve
```

### L3: 验证者
```
人类：帮我修这个 bug
Agent：分析 → 修复 → 验证 → 测试 → 报告
人类：review PR
```

### L4: 提交者
```
人类：帮我修这个 bug
Agent：分析 → 修复 → 测试 → 打开 PR
人类：merge
```

### L5: 合并者
```
人类：帮我修这个 bug
Agent：分析 → 修复 → 测试 → 打开 PR → 自动合并
人类：收到通知
```

## 信任边界

| 操作 | L1-L3 | L4 | L5+ |
|------|-------|-----|-----|
| 读文件 | ✅ | ✅ | ✅ |
| 写文件 | ✅ | ✅ | ✅ |
| 删除文件 | ❌ | ❌ | ❌ |
| 网络请求 | ❌ | ✅ | ✅ |
| 打开 PR | ❌ | ✅ | ✅ |
| 合并 PR | ❌ | ❌ | ✅ |
| 删除资源 | ❌ | ❌ | ❌ |

---

## 等级升级路径

```
L1 → L2 → L3 → L4 → L5
│    │    │    │    │
│    │    │    │    └── CI 成熟后
│    │    │    └── 测试覆盖 > 80%
│    │    └── Harness 稳定运行
│    └── Agent 错误率 < 10%
└── 项目初始化
```

---

## 定期 Review

每月检查：

1. Agent 自主等级是否合适
2. 错误率是否在可接受范围
3. 是否有需要回退的情况
4. Harness 是否需要调整
