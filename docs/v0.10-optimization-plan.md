# Nuwax Agent v0.10 优化计划

> 制定日期：2026-04-11
> 状态：进行中
> 目标版本：v0.10.0
> 定位：**新功能驱动**——三大核心能力升级

---

## 核心目标

v0.10 围绕用户反馈的三个核心目标展开：

1. **独立自主 Agent 方案** — 为政企合规做准备（本地化、审计、权限管控、Air-Gapped）
2. **Windows 客户端稳定性** — 服务崩溃自动恢复、进程管理加固、企业代理支持
3. **响应消息可交互确认** — 权限、问询、模型切换等操作内联于 chat 流，用户可逐步确认，与沙箱权限系统整合

**优先级: High（其他 T1 任务的基础）**

```typescript

interface ComplianceConfig {
```

### T1.2 — API Key 加密存储 ★ Sprint 1
**优先级: High（安全）**

当前 `db.ts::writeSetting` 以明文 JSON 存储 API keys。使用 Electron 内置 `safeStorage` API（Electron 15+ 可用，当前为 40）进行 OS 级加密：
- macOS：Keychain
- Windows：DPAPI
- Linux：SecretService/kwallet

核心改动：
- 添加 `writeEncryptedSetting` / `readEncryptedSetting` 对敏感 key 启用加密
- 首次启动检测未加密值，原地迁移，用 `settings_encryption_migrated` flag 防止重复迁移

**敏感 Key 列表：** `anthropic_api_key`、`agent_api_key`

**修改文件：**
- `src/main/db.ts`（添加加密读写函数 + 迁移逻辑）
- `src/main/bootstrap/migrate.ts`（添加加密迁移步骤 `migrateApiKeysEncryption`）
- `src/main/ipc/settingsHandlers.ts`（敏感 key 使用加密读写）

## Track 2 — Windows 客户端稳定性

### T2.1 — ManagedProcess 自动重启 ★ Sprint 1
**优先级: High（基础）**

扩展 `processManager.ts` 的 `ManagedProcess`，添加指数退避重启策略：

```typescript
interface RestartPolicy {
  enabled: boolean;
  maxRestarts: number;     // 默认 5
  backoffBase: number;     // ms，默认 1000
  backoffMax: number;      // ms，默认 30000
  resetAfterMs: number;    // 稳定运行后重置重启计数
}
```

关键设计：`stop()` / `kill()` 先将 `this.process = null` 再发信号，意外退出时 `this.process !== null` 为真才触发重启，避免主动停止误触发。

**修改文件：**
- `src/main/processManager.ts`
- `src/main/window/serviceManager.ts`（为 lanproxy、fileServer 配置重启策略）

## Track 3 — 响应消息可交互确认

### T3.1 — AcpEngine 交互式权限门控 ★ Sprint 1
**优先级: High（Track 3 核心）**

**关键发现：** `acpEngine.ts::handlePermissionRequest()` 当前直接返回 `allow_always` / `allow_once`，而 `pendingPermissions` Map（已定义）**从未被使用**——这正是需要接入的脚手架。

核心修改：
```typescript
// 将同步返回改为 Promise，挂起直到用户响应
return new Promise<AcpPermissionResponse>((resolve) => {
  this.pendingPermissions.set(params.toolCall.toolCallId, {
    resolve, options: params.options,
  });
  this.emit("permission.updated", { sessionId, requestId, toolCall, options });
});
```

新增 `respondPermission(requestId, response)` 方法，配置超时（默认 60s），超时后自动拒绝。

**修改文件：**
- `src/main/services/engines/acp/acpEngine.ts`（核心改动）
- `src/main/ipc/agentHandlers.ts`（添加 `agent:respondPermission`）
- `src/main/ipc/eventForwarders.ts`（确保 `permission.updated` 被转发）
