---

## 背景

NuwaClaw v0.10 围绕用户反馈的三个核心目标展开：

1. **独立自主 Agent 方案** — 为政企合规做准备（本地化、审计、权限管控、Air-Gapped）
2. **Windows 客户端稳定性** — 服务崩溃自动恢复、进程管理加固、企业代理支持
3. **响应消息可交互确认** — 权限、问询、模型切换等操作内联于 chat 流，用户可逐步确认

### 沙箱 + ACP 联动改造

沙箱机制变化会同步影响 ACP 协议，两者需要一起规划：

| 沙箱变化 | ACP 影响 |
|----------|----------|
| 权限模型调整 | `permission_request` 响应逻辑 |
| 交互式确认 | `question` 类型工具调用 |
| 审计日志 | 工具调用事件记录 |

### 任务 1.3 — 沙箱 autoFallback 语义修复

**优先级**: 🔴 高
**工作量**: 1h

**问题**: `autoFallback=startup-only` 应该拒绝会话，而非仅警告

**当前代码**（`acpEngine.ts:546-553`）:
```typescript
if (resolved.degraded && policy.autoFallback === "startup-only") {
  log.warn(...);  // 应该抛出错误
}
```

**期望行为**:
```typescript
if (resolved.degraded && policy.autoFallback === "startup-only") {
  throw new SandboxError(
    SandboxErrorCode.SANDBOX_UNAVAILABLE,
    "Sandbox unavailable with startup-only policy"
  );
}
```

**待完成**:
1. 修改 `acpEngine.ts` 中 `startup-only` 分支
2. 抛出 `SANDBOX_UNAVAILABLE` 错误
3. 前端捕获并展示错误提示
4. 更新 `docs/sandbox-plan.md`

**修改文件**:
- `src/main/services/engines/acp/acpEngine.ts`
- `src/renderer/App.tsx`
- `docs/sandbox-plan.md`

**验收标准**:
- `startup-only` 模式下沙箱不可用时会话被拒绝
