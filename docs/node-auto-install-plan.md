# 客户端启动时自动安装 Node 的实现方案（生产/测试/开发）

## 目标与范围
- 目标：当用户本地未安装 Node.js 时，客户端在启动阶段自动完成安装，并在后续正常拉起前端/脚本能力。
- 范围：生产/测试/开发环境均触发自动安装。
- 平台：Windows、macOS、Linux（x64/arm64）。

## 设计原则
- 最小侵入：不影响已有启动流程与用户路径。
- 可观测：日志可追踪、失败可定位。
- 可恢复：失败后可提示重试，并允许用户手动指定 Node 路径。
- 安全合规：校验下载产物完整性，避免执行未知二进制。

## 关键策略

### 1. 运行时检测
- 启动阶段优先检测 `node` 可执行文件是否可用：
  - 先检查环境变量 PATH 中的 `node`。
  - 再检查应用自带目录（例如 `app_data/node/`）。
- 如果检测到版本不满足最低要求（`>=22.x`），视为未安装。

### 2. 环境开关
- 生产/测试/开发环境均启用自动安装逻辑：
  - 通过编译期常量、构建产物标识或运行时配置判断（例如 `APP_ENV=production|test|development`）。
- 若需要临时关闭，可通过运行时配置提供 `AUTO_INSTALL_NODE=false`。

### 3. 安装策略（多平台）
- 采用官方 Node.js 发行包：
  - Windows：`zip` 或 `msi`（推荐 `zip` 解压后使用，避免弹系统安装器）。
  - macOS / Linux：`tar.xz`。
- 直接解压到应用私有目录（不写入系统目录）：
  - 示例路径：
    - Windows: `%AppData%/<app>/node/`
    - macOS: `~/Library/Application Support/<app>/node/`
    - Linux: `~/.local/share/<app>/node/`
- 安装后更新应用内部的 Node 路径配置，不修改系统 PATH。

### 4. 版本与架构选择
- 自动选择与客户端一致的架构（x64/arm64）。
- 仅下载 LTS 版本（满足 `>=22.x`），确保长期稳定。
- 推荐维护一份映射表（平台 × 架构 → 下载 URL）。

### 5. 安全与校验
- 下载后校验 SHA256 / GPG 签名（优先 SHA256）：
  - 校验文件需从官方来源获取并固定版本。
- 下载使用 HTTPS，避免中间人攻击。

### 6. 失败处理
- 如果下载或校验失败：
  - 给出清晰提示与重试入口。
  - 提供“手动指定 Node 路径”选项。
- 如果安装失败：
  - 清理残留目录，避免半安装状态。

## 启动流程示意
1. 客户端启动
2. 检测 Node 是否可用
3. 若不可用且为生产/测试/开发环境 → 触发自动安装
4. 校验 + 解压
5. 更新 Node 路径配置
6. 继续后续业务流程

## 模块划分建议
- NodeDetector：负责检测可用 Node 版本与路径
- NodeInstaller：负责下载、校验、解压
- NodeRegistry：负责保存与读取 Node 路径配置
- StartupGuard：启动流程接入点

## 风险与对策
- 下载速度慢/失败：支持断点续传或分段下载；提供手动安装指引。
- 平台差异导致解压失败：统一使用内置解压库或经过验证的跨平台解压方案。
- 安全风险：强制校验下载产物，禁止执行未校验文件。

## 可选增强
- 缓存已下载的 Node 包，避免重复下载。
- 支持企业内网镜像（配置下载源）。
- 允许用户禁用自动安装（高级设置）。

## 验收标准
- 生产环境首次启动可自动完成 Node 安装。
- 断网/下载失败时有明确提示。
- 多平台（Windows/macOS/Linux）均通过验证。
- 无需系统管理员权限。
