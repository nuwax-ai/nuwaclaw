# 侧边栏重构方案

## 目标

1. **移除「客户端」一级 Tab** — 其核心按钮功能提升到一级菜单
2. **一级菜单变为操作命令**，不再是页面导航：
   - 「开始会话」→ 触发登录/跳转 sessions + 启动
   - 「退出登录」→ 触发登出
   - 「会话」「任务」「应用开发」→ 跳转到对应 Tab
3. **二级菜单**（设置/依赖/授权/日志/关于）通过「设置」展开

## 实现方式

### 核心思路

侧边栏菜单的 `onClick` 不再只是 `setActiveTab`，而是根据 key 执行不同操作：

| 菜单 key | 操作 |
|----------|------|
| `start-session` | 若未登录 → 弹出登录表单；若已登录 → `setActiveTab("sessions")` + 触发开始会话 |
| `sessions` | `setActiveTab("sessions")` |
| `tasks` | `setActiveTab("tasks")` |
| `appdev` | `setActiveTab("appdev")` |
| `settings` | 展开二级 |
| `dependencies` | `setActiveTab("dependencies")` |
| `permissions` | `setActiveTab("permissions")` |
| `logs` | `setActiveTab("logs")` |
| `about` | `setActiveTab("about")` |

### Step 2 — 修改 Menu 的 onClick 逻辑

**文件：** `crates/agent-electron-client/src/renderer/App.tsx`

```typescript
const handleMenuClick = ({ key }: { key: string }) => {
  switch (key) {
    case "start-session":
      // 复用 ClientPage 的登录状态和开始会话逻辑
      if (!authState.isLoggedIn) {
        // 弹出登录表单（复用 ClientPage 的 renderLoginForm）
        // 或切换到 client tab
        setActiveTab("client");
      } else {
        // 已登录 → 跳转 sessions 并触发开始会话
        setActiveTab("sessions");
        // 通过 ref 或事件通知 SessionsPage 触发 start session
        sessionsPageRef.current?.startNewSession?.();
      }
      break;
    case "sessions":
    case "tasks":
    case "appdev":
    case "settings":
    case "dependencies":
    case "permissions":
    case "logs":
    case "about":
      setActiveTab(key as TabKey);
      break;
  }
};
```

### Step 3 — 登录表单处理

有两种方案：

**方案 A**：保留 `client` Tab 作为登录表单展示（当未登录时侧边栏的「开始会话」跳转到 `client` Tab）

**方案 B**：在侧边栏区域或 Modal 中内嵌登录表单

推荐 **方案 A** — 最小改动：保留 `client` Tab 用于登录/状态展示，但不在侧边栏菜单中显示「客户端」入口。

## 关键问题

### Q: 「开始会话」如何触发 SessionsPage 的 start session？

需要通过 ref 或事件传递。有两种方式：
1. `SessionsPage` 暴露 `startNewSession` 方法，父组件通过 ref 调用
2. 通过 `sessionsAutoOpen` state 触发（现有机制已支持）

推荐方式 2 — 复用现有 `sessionsAutoOpen` + `onAutoOpenConsumed` 机制：

```typescript
case "start-session":
  if (!authState.isLoggedIn) {
    setActiveTab("client"); // 跳转到 client tab 显示登录表单
  } else {
    setSessionsAutoOpen(true); // 触发 SessionsPage 自动开始会话
    setActiveTab("sessions");
  }
  break;
```

## 风险与权衡

- **「客户端」Tab 是否保留**：推荐保留作为登录表单展示页，但侧边栏不显示入口
- **操作 vs 导航混合**：侧边栏现在混合了操作命令和页面导航，用户需要理解「开始会话」是操作而非页面
- **向后兼容**：现有 Tab 渲染逻辑不变，只重组 menuItems
