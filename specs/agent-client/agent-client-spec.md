## 1. 功能需求总览

### 1.1 开发阶段划分

本项目采用**分阶段开发**策略，明确各阶段目标和范围：

#### 阶段一：核心通信与 UI（当前优先级）

**目标**：验证 `agent-server-admin` 与 `agent-client` 之间的双向通信能力，确保 `data-server` 的 P2P/TCP/WebSocket 通信正常工作。

| 功能模块 | 说明 | 优先级 |
|----------|------|--------|
| 基础运行能力 | 跨平台运行、系统托盘、开机自启动 | P0 |
| 连接管理| 连接 data-server、P2P/Relay 模式、心跳保活、重连 | P0 |
| 客户端信息 | 显示 ID、显示/隐藏密码、复制功能、修改密码 | P0 |
| 设置界面 | data-server 地址配置 | P0 |
| 权限管理 | 屏幕录制、辅助功能权限检测与引导 | P0 |
| 依赖管理 | Node.js 自动安装、npm 工具安装 | P0 |
| 跨平台打包 | Windows/macOS/Linux 安装包 | P0 |

**验证指标**：
- [ ] agent-client 可在 macOS/Windows/Linux 成功打包安装
- [ ] agent-client 能成功连接 data-server（hbbs/hbbr）
- [ ] agent-server-admin 能发现并连接 agent-client
- [ ] 双向通信正常（消息可发送/接收）
- [ ] 支持 P2P 直连（局域网）
- [ ] 支持 Relay 中继（跨公网）
- [ ] 支持 WebSocket 兼容模式（防火墙环境）

#### 阶段二：Agent 运行时集成（后续开发）

**目标**：集成 agent_runner，实现 Agent 任务执行、进度回传等功能。

| 功能模块 | 说明 | 优先级 |
|----------|------|--------|
| Agent 执行 | 任务执行、输出回传、任务取消 | P1 |
| Agent 状态 | 状态栏显示 agent 进程运行状态 | P1 |
| 远程桌面 | 屏幕捕获、输入模拟 | P1 |
| 文件传输 | 发送/接收文件 | P1 |
| 日志调试 | 开发者模式日志、导出日志 | P1 |
| 客户端升级 | 检查更新、下载安装 | P2 |
| 聊天界面 | Agent 聊天 UI | P2 |

#### 2.2.4 状态定义

```rust
/// 客户端信息状态
struct ClientInfoState {
    /// 客户端 ID（只读）
    client_id: String,

    /// 密码（掩码状态）
    password_masked: bool,
}

/// 密码修改状态（在设置 Tab - 安全子页面中管理）
struct PasswordChangeState {
    /// 原密码
    old_password: String,

    /// 新密码
    new_password: String,

    /// 确认密码
    confirm_password: String,

    /// 密码强度
    strength: PasswordStrength,

    /// 修改状态
    status: PasswordChangeStatus,
}

enum PasswordStrength {
    Weak,      // < 8 字符，红色
    Medium,    // >= 8 字符，无特殊字符，黄色
    Strong,    // >= 8 字符 + 特殊字符，绿色
}

enum PasswordChangeStatus {
    Idle,
    Validating,      // 验证原密码
    Changing,        // 修改中
    Success,         // 修改成功
    Error(String),   // 错误信息
}
```

**验证规则**：
1. 原密码必填，调用服务器验证
2. 新密码长度 >= 8 位
3. 新密码必须包含字母和数字（可选：特殊字符）
4. 两次新密码输入必须一致
5. 新密码不能与原密码相同

**验证时机**：
- 实时验证：输入时检查格式
- 提交时验证：再次检查所有规则

**服务器配置逻辑**：

| 配置项 | 说明 | 默认值 | 验证规则 |
|--------|------|--------|----------|
| 信令服务器地址 | hbbs 主机地址或域名 | `127.0.0.1` | 非空，支持域名和 IP |
| 信令服务器端口 | hbbs 监听端口 | `21116` | 1-65535 |
| 中继服务器地址 | hbbr 主机地址或域名 | `127.0.0.1` | 非空，支持域名和 IP |
| 中继服务器端口 | hbbr 监听端口 | `21117` | 1-65535 |
| API 服务器地址 | API 服务主机地址 | `127.0.0.1` | 非空，支持域名和 IP |
| API 服务器端口 | API 服务监听端口 | `8080` | 1-65535 |

**连接测试逻辑**：

1. 用户输入服务器地址后，可点击"连接测试"按钮验证连通性
2. 测试顺序：
   - 测试 hbbs 连接（TCP）
   - 测试 hbbr 连接（TCP）
   - 测试 API 服务器连接（HTTP）
3. 测试结果反馈：
   - ✅ 全部成功：显示"所有服务器连接正常"
   - ❌ 部分失败：显示失败的服务器列表及错误原因
   - 超时：显示"连接超时，请检查网络或服务器地址"

- 配置文件路径：`~/Library/Application Support/nuwax-agent/config/server.toml` (macOS)

- Linux: `~/.local/share/nuwax-agent/config/server.toml`

```toml
# server.toml
[hbbs]
host = "hbbs.example.com"
port = 21116

[hbbr]
host = "relay.example.com"
port = 21117

[api]
host = "api.example.com"
port = 8080
```

#### 3.2.6 rcoder/agent_runner（直接调用，不使用 gRPC）

**使用方式**：客户端不运行 gRPC 服务，直接通过 path 依赖调用 agent_runner 的函数接口。

```rust
use agent_runner::{
    start_agent, cancel_agent, get_agent_status,
    AgentRequest, AgentOutput, AgentStatus,
};

// 1. 启动 Agent 任务（返回异步迭代器，流式输出）
let mut output_stream = start_agent(AgentRequest {
    prompt: "帮助我写代码".to_string(),
    work_dir: PathBuf::from("./workspace"),
    env: vec![("KEY".to_string(), "value".to_string())],
    timeout: Some(Duration::from_secs(300)),
}).await?;

// 2. 处理输出流
while let Some(output) = output_stream.next().await {
    match output.output_type {
        // 标准输出
        AgentOutputType::Stdout => {
            println!("{}", output.content);
        }
        // 标准错误
        AgentOutputType::Stderr => {
            eprintln!("[错误] {}", output.content);
        }
        // 状态更新
        AgentOutputType::Status => {
            println!("[状态] {}", output.content);
        }
        // 执行完成
        AgentOutputType::Complete => {
            println!("[完成] {}", output.content);
        }
        // 执行错误
        AgentOutputType::Error => {
            eprintln!("[执行错误] {}", output.content);
        }
    }
}

// 3. 取消任务
cancel_agent(session_id).await?;

// 4. 查询状态
let status = get_agent_status(session_id).await?;
match status {
    AgentStatus::Idle => println!("空闲"),
    AgentStatus::Active => println!("执行中"),
    AgentStatus::Pending => println!("等待中"),
    AgentStatus::Terminating => println!("停止中"),
    AgentStatus::Error => println!("出错"),
}
```

**核心函数接口**：

| 函数 | 返回类型 | 说明 |
|------|----------|------|
| `start_agent(request)` | `AsyncIter<AgentOutput>` | 启动任务，流式返回输出 |
| `cancel_agent(session_id)` | `Result<()>` | 取消正在执行的任务 |
| `get_agent_status(session_id)` | `Result<AgentStatus>` | 查询任务状态 |
| `list_active_agents()` | `Vec<SessionInfo>` | 列出所有活跃任务 |

## 5. 通信协议设计

### 5.1 Protobuf 消息类型

| 消息 ID | 类型名称 | 用途 |
|---------|----------|------|
| 100 | HandshakeRequest | 握手请求 |
| 101 | HandshakeResponse | 握手响应 |
| 102 | HeartbeatPing | 心跳 |
| 103 | HeartbeatPong | 心跳响应 |
| 200 | AgentTaskRequest | Agent 任务请求 |
| 201 | AgentTaskResponse | Agent 任务响应 |
| 202 | AgentTaskOutput | Agent 任务输出 |
| 203 | AgentTaskCancel | 取消任务 |
| 300 | FileTransferRequest | 文件传输请求 |
| 301 | FileTransferChunk | 文件分片 |
| 302 | FileTransferComplete | 传输完成 |
| 400 | ScreenCaptureRequest | 屏幕捕获请求 |
| 401 | ScreenCaptureFrame | 屏幕帧数据 |
| 402 | InputEvent | 输入事件 |
| 500 | UpdatePasswordRequest | 更新密码请求 |
| 501 | UpdatePasswordResponse | 更新密码响应 |

### 5.2 连接流程

```
1. 客户端 → data-server: HandshakeRequest
   - 包含客户端信息（OS 版本、应用版本等）
   - 包含协议版本

2. data-server → 客户端: HandshakeResponse
   - 成功: 包含分配的 Client ID、连接模式
   - 失败: 包含错误信息

3. 双方保持心跳 (间隔 5 秒)
   - HeartbeatPing → HeartbeatPong
   - 包含延迟信息

4. 传输数据
   - 根据消息类型路由到对应处理器
```

### 5.3 完整消息流转流程（Browser → Agent 执行）

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                              完整消息流转流程                                         │
└─────────────────────────────────────────────────────────────────────────────────────┘

  浏览器                    agent-server-admin                  data-server          agent-client
    │                            │                                 │                    │
    │  1. 用户输入 Prompt         │                                 │                    │
    │ ──────────────────────────>│                                 │                    │
    │                            │                                 │                    │
    │                            │  2. 通过 WebSocket 发送到 data-server │
    │                            │────────────────────────────────>│                    │
    │                            │                                 │                    │
    │                            │                                 │  3. 路由到目标客户端
    │                            │                                 │───────────────────>│
    │                            │                                 │                    │
    │                            │                                 │                    │  4. Agent 执行
    │                            │                                 │                    │───> Agent Runner
    │                            │                                 │                    │
    │                            │                                 │  5. 输出回传       │
    │                            │                                 │<───────────────────│
    │                            │                                 │                    │
    │  6. SSE 推送结果            │  6. SSE 推送结果                 │                    │
    │<───────────────────────────│────────────────────────────────│                    │
    │                            │                                 │                    │
    │  7. 浏览器实时显示          │                                 │                    │
    │    (流式输出)              │                                 │                    │
    │                            │                                 │                    │
```

**详细步骤说明**：

| 步骤 | 发送方 | 接收方 | 消息类型 | 说明 |
|------|--------|--------|----------|------|
| 1 | 浏览器 | agent-server-admin | HTTP POST | 用户提交 prompt |
| 2 | agent-server-admin | data-server | WebSocket + Protobuf | 转发任务请求 |
| 3 | data-server | agent-client | 路由转发 + Protobuf | 根据 Client ID 定位目标 |
| 4 | agent-client | agent_runner | 直接调用 | 解析 protobuf，调用内部逻辑执行 |
| 5 | agent-client | data-server | WebSocket + Protobuf | 流式输出回传 |
| 6 | data-server | agent-server-admin | WebSocket + Protobuf | 转发输出 |
| 7 | agent-server-admin | 浏览器 | SSE | 流式推送结果 |
| 8 | 浏览器 | 用户 | 渲染 | 实时显示执行过程 |

### 5.4 客户端 Agent 执行机制

**设计原则**：客户端不运行 gRPC 服务端，直接解析 protobuf 消息后调用 agent_runner 内部逻辑。

```
agent-client 内部架构
┌─────────────────────────────────────────────────────────────────┐
│                         agent-client                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────┐    ┌─────────────────────┐            │
│  │  Protobuf 消息解析   │───▶│  消息路由器          │            │
│  │  (消息反序列化)      │    │  (根据 message_id)  │            │
│  └─────────────────────┘    └──────────┬──────────┘            │
│                                        │                        │
│                                        ▼                        │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                     任务处理器                           │   │
│  │  ┌─────────────────────────────────────────────────┐   │   │
│  │  │  ChatHandler (对应 gRPC Chat)                   │   │   │
```

#### 5.4.4 完整的 Chat 处理流程（客户端直接调用）

```rust
use agent_runner::{
    start_agent, AgentRequest, AgentStatus,
    AGENT_REGISTRY, SESSION_CACHE,
};
use crate::message_converter::{to_agent_request, to_chat_response};

pub async fn handle_chat(grpc_request: ChatRequest) -> Result<ChatResponse, Status> {
    let project_id = grpc_request.project_id;
    let session_id = grpc_request.session_id;

    // 1. 检查 Agent 状态（防止并发）
    if let Some(info) = AGENT_REGISTRY.get_agent_info(&project_id) {
        if info.status == AgentStatus::Active || info.status == AgentStatus::Pending {
            return Ok(ChatResponse {
                success: false,
                error_code: Some("ERR_AGENT_BUSY".to_string()),
                error: Some("Agent 正在执行任务，请等待当前任务完成".to_string()),
                ..Default::default()
            });
        }
    }

    // 2. 预注册 Pending 状态
    AGENT_REGISTRY.set_pending(&project_id);

    // 3. 转换 gRPC 请求为 AgentRequest
    let agent_request = to_agent_request(grpc_request);

    // 4. 启动任务（返回异步迭代器）
    let mut output_stream = start_agent(agent_request).await?;

    // 5. 处理输出流
    let mut session_id = None;
    while let Some(output) = output_stream.next().await {
        // 发送到 SESSION_CACHE（供进度订阅使用）
        if let Some(sid) = &output.session_id {
            session_id = Some(sid.clone());
            SESSION_CACHE.push_message(sid, output.clone());
        }
    }

    // 6. 构建响应
    Ok(to_chat_response(&project_id, session_id))
}
```

#### 5.4.5 进度推送实现（对应 gRPC SubscribeProgress）

```rust
pub async fn subscribe_progress(session_id: String) -> impl Stream<Item = ProgressEvent> {
    let (tx, rx) = mpsc::channel(100);

    tokio::spawn(async move {
        // 从 SESSION_CACHE 订阅消息
        if let Some(session_data) = SESSION_CACHE.get(&session_id) {
            let (mut msg_rx, _) = session_data.create_new_connection(100).await;

            loop {
                tokio::select! {
                    // 接收进度消息
                    Some(msg) = msg_rx.recv() => {
                        let event = ProgressEvent {
                            message_type: format!("{:?}", msg.message_type),
                            sub_type: msg.sub_type,
                            payload: serde_json::to_string(&msg.data).unwrap_or_default(),
                            timestamp: msg.timestamp.timestamp_millis(),
                        };
                        if tx.send(Ok(event)).await.is_err() {
                            break; // 客户端断开
                        }
                    }
                    // 心跳保活（30秒）
                    _ = tokio::time::sleep(Duration::from_secs(30)) => {
                        let heartbeat = ProgressEvent {
                            message_type: "Heartbeat".to_string(),
                            sub_type: "ping".to_string(),
                            payload: r#"{"type":"heartbeat"}"#.to_string(),
                            timestamp: Utc::now().timestamp_millis(),
                        };
                        if tx.send(Ok(heartbeat)).await.is_err() {
                            break;
                        }
                    }
                }
            }
        }
    });

    ReceiverStream::new(rx)
}
```

#### 5.4.6 取消任务实现（对应 gRPC CancelSession）

```rust
pub async fn cancel_session(
    project_id: String,
    session_id: String,
    reason: String,
) -> Result<CancelResponse> {
    // 1. 查询 Agent 状态
    let agent_info = match AGENT_REGISTRY.get_agent_info(&project_id) {
        Some(info) => info,
        None => return Ok(CancelResponse {
            success: true,
            message: Some("任务不存在或已完成".to_string()),
        }),
    };

    // 2. 发送取消通知
    let cancel_tx = agent_info.cancel_tx.clone();
    let cancel_notification = CancelNotification::new(SessionId::new(session_id.as_str()));
    let (result_tx, result_rx) = oneshot::channel();

    cancel_tx.send(CancelNotificationRequestWrapper {
        cancel_notification,
        result_tx,
    })?;

    // 3. 等待取消结果（30秒超时）
    match tokio::time::timeout(Duration::from_secs(30), result_rx).await {
        Ok(Ok(result)) => {
            if result.is_success() {
                // 清理资源
                SESSION_CACHE.remove(&session_id);
                AGENT_REGISTRY.clear_pending_if_exists(&project_id);
            }
            Ok(CancelResponse {
                success: result.is_success(),
                message: result.message(),
            })
        }
        _ => Ok(CancelResponse {
            success: false,
            message: Some("取消超时".to_string()),
        }),
    }
}
```

#### 5.5.1 业务通道 API 设计

在 nuwax-rustdesk 中新增 `business_channel` 模块：

```rust
// vendors/nuwax-rustdesk/src/business_channel.rs

use std::sync::Arc;
use tokio::sync::{mpsc, RwLock};
use hbb_common::{ResultType, Stream};

/// 业务通道 ID（固定值）
pub const BUSINESS_CHANNEL_ID: u32 = 0xBIZ;

/// 业务消息类型
#[derive(Debug, Clone)]
pub enum BusinessMessageType {
    /// Agent 任务请求
    AgentTask = 0x01,
    /// Agent 任务响应
    AgentResponse = 0x02,
    /// Agent 进度推送
    AgentProgress = 0x03,
    /// Agent 取消请求
    AgentCancel = 0x04,
    /// 自定义业务消息
    Custom(u32),
}

/// 业务通道配置
#[derive(Clone)]
pub struct BusinessChannelConfig {
    /// 通道 ID（固定为 BUSINESS_CHANNEL_ID）
    pub channel_id: u32,
    /// 消息发送超时（毫秒）
    pub send_timeout_ms: u64,
    /// 消息接收缓冲区大小
    pub buffer_size: usize,
}

impl Default for BusinessChannelConfig {
    fn default() -> Self {
        Self {
            channel_id: BUSINESS_CHANNEL_ID,
            send_timeout_ms: 5000,
            buffer_size: 100,
        }
    }
}

/// 业务数据通道
#[derive(Clone)]
pub struct BusinessChannel {
    /// 底层 Stream
    stream: Arc<dyn Stream + Send + Sync>,
    /// 配置
    config: BusinessChannelConfig,
    /// 消息发送器
    sender: Arc<RwLock<Option<mpsc::UnboundedSender<Vec<u8>>>>>,
    /// 消息接收器（用于内部路由）
    receiver: Arc<RwLock<Option<mpsc::UnboundedReceiver<Vec<u8>>>>>,
}

impl BusinessChannel {
    /// 从底层 Stream 创建业务通道
    pub async fn from_stream(
        stream: Arc<dyn Stream + Send + Sync>,
        config: Option<BusinessChannelConfig>,
    ) -> ResultType<Self> {
        let config = config.unwrap_or_default();

        let (tx, rx) = mpsc::unbounded_channel(config.buffer_size);

        Ok(Self {
            stream,
            config,
            sender: Arc::new(RwLock::new(Some(tx))),
            receiver: Arc::new(RwLock::new(Some(rx))),
        })
    }

    /// 发送业务消息
    pub async fn send(&self, message_type: BusinessMessageType, payload: &[u8]) -> ResultType<()> {
        let mut sender = self.sender.write().await;
        if let Some(tx) = sender.as_ref() {
            // 封装消息
            let mut data = Vec::with_capacity(1 + 4 + payload.len());
            data.push(message_type as u8);
            data.extend_from_slice(&(payload.len() as u32).to_le_bytes());
            data.extend_from_slice(payload);

            tx.send(data)?;
            Ok(())
        } else {
            Err(anyhow::anyhow!("Business channel sender not initialized"))
        }
    }

    /// 接收业务消息
    pub async fn recv(&self) -> ResultType<Option<(BusinessMessageType, Vec<u8>)>> {
        let mut receiver = self.receiver.write().await;
        if let Some(rx) = receiver.as_ref() {
            match rx.recv().await {
                Some(data) => {
                    if data.len() < 5 {
                        return Err(anyhow::anyhow!("Invalid message format"));
                    }
                    let msg_type = BusinessMessageType::Custom(data[0] as u32);
                    let len = u32::from_le_bytes([data[1], data[2], data[3], data[4]]) as usize;
                    let payload = if data.len() > 5 {
                        data[5..5 + len].to_vec()
                    } else {
                        Vec::new()
                    };
                    Ok(Some((msg_type, payload)))
                }
                None => Ok(None),
            }
        } else {
            Err(anyhow::anyhow!("Business channel receiver not initialized"))
        }
    }

    /// 订阅业务消息流
    pub fn subscribe(&self) -> mpsc::UnboundedReceiver<(BusinessMessageType, Vec<u8>)> {
        let (tx, rx) = mpsc::unbounded_channel(self.config.buffer_size);
        let mut sender = self.sender.write().await;

        // 创建一个内部任务来转发消息
        let mut inner_rx = {
            let mut receiver = self.receiver.write().await;
            receiver.take()
        };

        if let Some(inner_rx_val) = inner_rx {
            tokio::spawn(async move {
                let mut tx = tx;
                let mut inner_rx = inner_rx_val;
                while let Some(data) = inner_rx.recv().await {
                    if data.len() < 5 {
                        continue;
                    }
                    let msg_type = BusinessMessageType::Custom(data[0] as u32);
                    let len = u32::from_le_bytes([data[1], data[2], data[3], data[4]]) as usize;
                    let payload = if data.len() > 5 {
                        data[5..5 + len].to_vec()
                    } else {
                        Vec::new()
                    };
                    if tx.send((msg_type, payload)).is_err() {
                        break;
                    }
                }
            });
        }

        rx
    }

    /// 检查通道是否有效
    pub fn is_valid(&self) -> bool {
        self.sender.read().await.is_some()
    }
}
```

```rust
use crate::message_converter::{to_agent_request, to_agent_response};

/// 业务通道管理器
pub struct BusinessChannelManager {

    channel: Option<BusinessChannel>,
    /// 配置
    config: BusinessChannelConfig,

impl BusinessChannelManager {
    pub fn new() -> Self {
        Self {
            channel: None,
            config: BusinessChannelConfig::default(),
        }
    }

    /// 从 nuwax-rustdesk 的 Stream 创建业务通道
    pub async fn create_from_stream(&mut self, stream: Arc<dyn Stream + Send + Sync>) -> Result<()> {
        self.channel = Some(BusinessChannel::from_stream(stream, Some(self.config.clone())).await?);
        Ok(())
    }

    /// 发送 Agent 任务请求
    pub async fn send_agent_task(&self, task: AgentTaskRequest) -> Result<()> {

            let payload = to_agent_request_payload(task);
            channel.send(BusinessMessageType::AgentTask, &payload).await?;
        }
        Ok(())
    }

    /// 接收业务消息并处理
    pub async fn handle_messages(&self) -> Result<()> {
        if let Some(channel) = &self.channel {
            let mut subscriber = channel.subscribe();
            while let Some((msg_type, payload)) = subscriber.recv().await {
                match msg_type {
                    BusinessMessageType::AgentTask => {
                        self.handle_agent_task(payload).await?;
                    }
                    BusinessMessageType::AgentCancel => {
                        self.handle_cancel(payload).await?;
                    }
                    _ => {
                        warn!("Unknown business message type: {:?}", msg_type);
                    }
                }
            }
        }
        Ok(())
    }

    /// 处理 Agent 任务
    async fn handle_agent_task(&self, payload: Vec<u8>) -> Result<()> {
        let task: AgentTaskPayload = protobuf::decode(&payload)?;
        let request = to_agent_request(task);

        // 调用 agent_runner 执行任务
        let mut output_stream = start_agent(request).await?;

        // 订阅输出并发送进度
        while let Some(output) = output_stream.next().await {
            let progress = to_progress_payload(&output);
            if let Some(channel) = &self.channel {
                channel.send(BusinessMessageType::AgentProgress, &progress).await?;
            }
        }

        Ok(())
    }
```

```rust
struct I18nManager {
    /// 当前语言
    current_lang: RwLock<Language>,
    /// 语言资源
    resources: DashMap<Language, ResourceBundle>,
    /// 回退资源
    fallback: ResourceBundle,
}

impl I18nManager {
    /// 加载语言资源
    async fn load_language(&self, lang: Language) -> Result<()> {
        let resource = ResourceBundle::from_file(format!("i18n/{}/messages.toml", lang.code())).await?;
        self.resources.insert(lang, resource);
        Ok(())
    }

    /// 获取翻译字符串
    fn t(&self, key: &str) -> String {
        let lang = *self.current_lang.read();
        self.resources.get(&lang)
            .or(Some(&self.fallback))
            .and_then(|r| r.get(key))
            .unwrap_or_else(|| key.to_string())
    }

    /// 格式化带参数的字符串
    fn t_with_args(&self, key: &str, args: &[(&str, &str)]) -> String {
        let mut result = self.t(key);
        for (name, value) in args {
            result = result.replace(&format!("{{{name}}}"), value);
        }
        result
    }
}
```

```rust
    fn format_date(&self, date: &DateTime<Utc>) -> String {
        match self.locale {
            Language::ZhCN => date.format("%Y年%m月%d日").to_string(),
            Language::JaJP => date.format("%Y年%m月%d日").to_string(),
            _ => date.format("%B %d, %Y").to_string(),
        }

    /// 格式化时间
    fn format_time(&self, time: &DateTime<Utc>) -> String {
        match self.locale {
            Language::ZhCN => time.format("%H:%M:%S").to_string(),
            Language::JaJP => time.format("%H:%M:%S").to_string(),
            _ => time.format("%H:%M:%S").to_string(),
        }
    }

    /// 格式化文件大小
    fn format_file_size(&self, bytes: u64) -> String {
        const UNITS: [&str; 5] = ["B", "KB", "MB", "GB", "TB"];
        let mut size = bytes as f64;
        let mut unit_index = 0;

        while size >= 1024.0 && unit_index < UNITS.len() - 1 {
            size /= 1024.0;
            unit_index += 1;
        }

        match self.locale {
            Language::ZhCN => format!("{:.1} {}", size, UNITS[unit_index]),
            _ => format!("{} {:.1} {}", size, UNITS[unit_index]),
        }
    }
}
```

#### 5.13.3 并发控制

```rust
/// 并发任务管理器
struct ConcurrencyManager {
    /// 最大并发任务数
    max_concurrent: usize,
    /// 当前运行的任务
    running_tasks: Arc<RwLock<HashMap<String, AgentSessionState>>>,
    /// 等待队列
    waiting_queue: Arc<RwLock<Vec<TaskRequest>>>,
}

impl ConcurrencyManager {
    /// 提交任务
    async fn submit(&self, request: TaskRequest) -> Result<SubmissionResult> {
        let running_count = self.running_tasks.read().len();

        if running_count >= self.max_concurrent {
            // 加入等待队列
            self.waiting_queue.write().push(request);
            Ok(SubmissionResult::Queued)
        } else {
            // 直接执行
            self.start_task(request).await?;
            Ok(SubmissionResult::Accepted)
        }
    }

    /// 任务完成回调
    fn on_task_completed(&self, task_id: &str) {
        // 从运行中移除
        self.running_tasks.write().remove(task_id);

        // 检查等待队列，启动下一个任务
        if let Some(next_task) = self.waiting_queue.write().pop() {
            tokio::spawn(async move {
                self.start_task(next_task).await;
            });
        }
    }
}
```

#### 5.13.4 僵尸会话检测与清理

```rust
/// 会话清理策略
struct SessionCleanupPolicy {
    /// 清理间隔
    cleanup_interval: Duration,
    /// 僵尸判定时间（超过此时间无活动视为僵尸）
    zombie_threshold: Duration,
    /// 保留历史记录时间
    history_retention: Duration,
}

impl SessionCleanupPolicy {
    /// 执行清理
    async fn cleanup(&self, sessions: &DashMap<String, AgentSessionState>) -> CleanupResult {
        let mut zombie_sessions = Vec::new();
        let now = Utc::now();

        for (session_id, session) in sessions.iter() {
            if let Some(last_activity) = self.get_last_activity(session).await {
                let elapsed = now.signed_duration_since(last_activity);
                if elapsed > self.zombie_threshold {
                    zombie_sessions.push(session_id.clone());
                }
            }
        }

        // 清理僵尸会话
        for session_id in &zombie_sessions {
            sessions.remove(session_id);
        }

        CleanupResult {
            cleaned_count: zombie_sessions.len(),
            timestamp: now,
        }
    }
}
```
